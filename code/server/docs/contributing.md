# Contribuindo

## Diretrizes Gerais
- Use branches descritivos (ex: `feature/nome`, `fix/bug`)
- Commits claros e objetivos (padrão Conventional Commits)
- Pull Requests com descrição do problema e solução

## Estilo de Código
- Python: PEP8, Black, isort, flake8
- Testes obrigatórios para novas features

## Revisão
- Sempre abra PRs para revisão
- Teste localmente antes de submeter
- Corrija conflitos de merge
